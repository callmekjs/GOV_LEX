# ADR-003: metadata를 자유 dict로 두고 점진 승격하는 정책

## Context
KR/US 법률·입법 API는 필드 구조와 네이밍이 상이하고, 같은 소스 내에서도 시기별 편차가 존재합니다.  
초기부터 metadata를 강한 고정 스키마로 묶으면 수집 실패와 누락이 증가할 가능성이 높았습니다.

## Decision
핵심 9필드는 고정하고, 국가/소스 특화 정보는 `metadata: dict[str, Any]`로 수용한 뒤 관찰 데이터를 기반으로 점진 승격합니다.  
즉, 초기에는 수용성 우선, 이후 반복 사용 키를 기준으로 서브모델/공통 필드로 승격합니다.

## Consequences
- 장점:
  - 신규 소스 연결 시 파이프라인 중단 없이 빠르게 흡수할 수 있습니다.
  - 어떤 필드가 실제 운영에 중요한지 데이터 기반으로 판단할 수 있습니다.
  - 스키마 진화 비용을 분산시켜 안정적 확장이 가능합니다.
- 단점:
  - 초기에 metadata 품질 편차가 커질 수 있습니다.
  - 팀 내 네이밍 규약이 없으면 의미 중복 키가 발생할 수 있습니다.

## Alternatives Considered
- 초기부터 국가별 metadata 고정 모델(KRMetadata/USMetadata) 강제:
  - 타입 안정성은 높지만 소스 변경 대응성이 떨어져 수집 속도를 저하시킬 수 있어 보류했습니다.
- metadata 필드 최소화(거의 미수집):
  - 단순성은 높지만 도메인 정보 손실이 커 Phase 3의 관계 추출/온톨로지 연계를 어렵게 하므로 채택하지 않았습니다.
