# ADR-007: BM25에서 벡터 검색으로 마이그레이션하는 시점과 트리거

## Context
현재 MVP는 BM25 중심의 검색/검증 흐름으로도 기본 정확 매칭 요구를 충족합니다.  
하지만 도메인 용어 동의어, 표현 차이, 긴 문맥 질의에서는 의미 기반 검색 필요성이 증가합니다.

## Decision
초기 기본 검색은 BM25를 유지하고, 아래 트리거가 충족될 때 벡터/하이브리드 검색으로 단계적으로 전환합니다.

전환 트리거:
1. 동의어/의미 유사 질의에서 BM25 재현율 저하가 반복 측정될 때
2. Phase 3-3 엔티티/관계 추출이 안정화되어 의미 검색 이득이 명확할 때
3. 운영 비용(임베딩 생성/저장/조회) 대비 성능 이점이 검증될 때
4. Phase 4 RAG 답변 품질 목표(정답률/인용 정확도)를 BM25 단독으로 충족하지 못할 때

## Consequences
- 장점:
  - 과도한 조기 최적화를 피하고, 검증 가능한 근거로 기술 전환이 가능합니다.
  - BM25의 단순성과 재현성을 유지하면서 고도화 여지를 확보합니다.
  - 챗봇 Phase 전환 시 기술 리스크를 단계적으로 관리할 수 있습니다.
- 단점:
  - 전환 전까지 의미 유사 검색 품질은 제한적일 수 있습니다.
  - 하이브리드 도입 시 인프라와 운영 복잡도가 증가합니다.

## Alternatives Considered
- 시작부터 벡터 검색 기본값 채택:
  - 기능적으로 매력적이지만 초기 비용/복잡도가 커 MVP 목표와 충돌해 보류했습니다.
- BM25만 고정 사용:
  - 운영은 단순하지만 Phase 4 RAG 품질 상한이 낮아질 가능성이 커 장기적으로 부적합하다고 판단했습니다.
