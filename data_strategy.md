# GovLex 데이터 전략 (10K 고품질 우선)

## 1) 전략 요약

기존의 대량 수집(30K~50K) 중심 전략을 조정한다.  
현재 우선순위는 **10K 내외의 고품질 Legal Corpus 확보**이며,  
10K 도달 이후에는 추가 수집보다 **분류·품질관리·검색 가능성 개선**에 집중한다.

> 1차 목표: **총 10K 문서 확보 + 품질 지표 체계 정착**

---

## 2) 왜 10K에서 품질로 전환하는가

- 데이터 규모가 커질수록 중복·결측·편향이 함께 증가할 수 있다.
- 포트폴리오 핵심은 단순 크기가 아니라 **신뢰 가능한 Corpus 운영 능력**이다.
- CODIT/거브테크 맥락에서 중요한 것은 수집량보다 **분류 정확도, QA 추적성, 재현성**이다.
- Phase 4 RAG에서도 문서 수보다 **검색 가능한 문서와 인용 가능한 근거 문서**가 더 중요하다.

---

## 3) 1차 수집 목표 (10K)

| 항목 | 목표 |
|---|---|
| 총 문서 수 | 약 10,000건 |
| 국가 비율 | KR 7,000 / US 3,000 |
| 핵심 도메인 | AI, 개인정보, 데이터, 플랫폼, 사이버보안, 핀테크 |
| 문서 유형 | law / bill 중심, decree 일부 포함 |

---

## 4) 수집 우선순위

### P0 (즉시)
- AI / 데이터 / 개인정보 / 플랫폼 / 사이버보안 / 핀테크

### P1 (다음 배치)
- 소비자보호 / 전자상거래 / 저작권 / 방송통신 / 디지털자산

### P2 (확장)
- 산업안전 / 헬스데이터 / 공공데이터 / 행정규칙·가이드라인

---

## 5) policy_area taxonomy (초안)

검색/필터/리포트 통일을 위해 `policy_area` 분류를 다음으로 고정한다.

1. `ai_governance`
2. `privacy_data_protection`
3. `cybersecurity`
4. `platform_online_safety`
5. `fintech_digital_asset`
6. `consumer_commerce`
7. `public_data_digital_gov`
8. `ip_content_media`
9. `other`

분류 규칙:
- 문서당 최소 1개 `policy_area`를 부여한다.
- 다중 주제 문서는 주(primary) 1개 + 보조(secondary) 태그를 허용한다.
- 초기에는 키워드 기반 분류, 이후 Golden Set으로 보정한다.

---

## 6) 데이터 품질 퍼널 지표 정의

수집 성과는 총량이 아니라 퍼널 지표로 관리한다.

| 지표 | 의미 |
|---|---|
| `raw_collected` | API/원천에서 가져온 전체 문서 수 |
| `normalized` | 공통 스키마 변환 성공 문서 수 |
| `qa_passed` | QA 룰(R01/R02/R05/R07) 통과 문서 수 |
| `search_usable` | 검색/RAG 인덱싱에 실제 사용 가능한 문서 수 |
| `cited_ready` | 출처 인용 품질까지 충족한 핵심 문서 수 |

권장 모니터링:
- `qa_passed / normalized` (목표 90% 이상)
- `search_usable / qa_passed`
- `cited_ready / search_usable`

---

## 7) 10K 이후 집중 작업

| 작업 | 목적 |
|---|---|
| `policy_area` 분류 정제 | 주제별 검색/분석 정확도 향상 |
| `source_type` 균형 점검 | law/bill/decree 편향 완화 |
| KR/US 비율 보정 | 국가 편중 완화 |
| QA 리포트 고도화 | 결측/중복/날짜 오류를 숫자로 관리 |
| `search_usable` 판정 로직 추가 | RAG 실사용 문서만 분리 관리 |
| Golden Set 100 구축 | 데모/평가/면접 공통 기준 |
| 실패 카탈로그 분석 | 오류 다발 소스·키워드 개선 |

---

## 8) Golden Set 100 구축 계획

목표:
- 면접 시연, RAG 평가, 회귀 테스트의 공통 기준 셋 확보

구성 원칙:
- KR 70 / US 30
- `law` 40 / `bill` 40 / `decree|minutes` 20
- P0 도메인 균형 분포
- 출처 URL/발행일/핵심 메타데이터가 명확한 문서 우선

운영:
1. 후보 300개 추출
2. 수동 검토로 100개 확정
3. 분기별 갱신(삭제/교체 이력 기록)

---

## 9) 운영 루틴

1. 키워드 배치 수집 실행
2. QA 리포트 확인 (`quality_report.md`)
3. 실패 카탈로그/패턴 확인 (`docs/failure_patterns.md`)
4. 대시보드 갱신 (`docs/dashboard.md`)
5. 월 1회 키워드/분류 리밸런싱

---

## 10) 성공 기준 (Phase 3~4 연결)

- 총 10K 내외 문서 확보
- `qa_passed / normalized` 90% 이상
- 필수 필드 누락률 1% 이하
- R07(날짜 파싱 실패) 추세 안정
- P0 도메인별 최소 문서 수 확보
- Golden Set 100 구축 완료
- 대시보드에서 국가/source_type/policy_area/QA 지표 확인 가능

---

## 11) 장기 확장 (보조 목표)

30K~50K 확장은 **품질 체계가 안정화된 이후**의 보조 목표로 둔다.  
즉, 대량 수집은 1차 목표가 아니라 품질관리 체계 완료 후의 2차 단계다.
